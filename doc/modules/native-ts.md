# native-ts 模块

## 模块定位

`src/native-ts` 存放以 TypeScript 形式实现的底层算法/原生能力替代件。它们通常偏“基础能力”而不是业务逻辑。

规模统计：

- 文件数：4
- 行数：4081
- 子目录：
  - `yoga-layout`，2 文件，2712 行
  - `color-diff`，1 文件，999 行
  - `file-index`，1 文件，370 行

## 模块职责

### 1. `yoga-layout`

这是最重的子模块，说明终端布局计算很可能在一定程度上复用了 yoga 风格的布局模型。

### 2. `color-diff`

提供颜色差异或终端色彩相关的基础算法，典型用途包括渲染对比、diff 可视化等。

### 3. `file-index`

提供文件索引相关能力，体量不大但属于性能/基础设施工具。

## 为什么这个目录单独存在

这类代码有两个特点：

- 与业务无关，但对运行时能力很重要
- 不适合散落进 `utils`

单独放在 `native-ts` 可以提醒读者：这里是“近底层算法实现”，不是 सामान्य helper。

## 与其他模块的关系

- `ink` / 布局系统：可能消费 `yoga-layout`
- `components/StructuredDiff` 等 UI：可能消费 `color-diff`
- 搜索/索引相关能力：可能消费 `file-index`
